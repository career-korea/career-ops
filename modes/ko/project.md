# 모드: project — 포트폴리오 프로젝트 평가

만들려는 포트폴리오 프로젝트 아이디어를 6개 차원으로 평가한다.

---

## 평가 매트릭스 (1-5점)

| 차원 | 가중치 | 5점 | 1점 |
|------|--------|-----|-----|
| 목표 직무 신호 | 25% | JD 핵심 스킬을 직접 증명 | 직무와 무관 |
| 독창성 | 20% | 차별화된 주제나 접근법 | 흔한 클론 프로젝트 |
| 데모 가능성 | 20% | 실제 서비스처럼 동작, 2분 시연 가능 | 코드만 있고 실행 불가 |
| 지표 가능성 | 15% | 명확한 측정값 (성능, 정확도, 비용 등) | 측정 불가 |
| MVP 기간 | 10% | 1~2주 이내 | 3개월 이상 |
| 스토리 가능성 | 10% | 기획 의도·트레이드오프·회고 풍부 | 구현만 있음 |

## 한국 취업 포트폴리오 기준

채용 담당자가 실제로 확인하는 것:

| 플랫폼 | 확인 포인트 |
|--------|-----------|
| **GitHub** | README 퀄리티, 커밋 이력, 코드 가독성, 협업 PR 이력 |
| **배포 URL** | 실제 접속 가능한 서비스 (Vercel, Railway, AWS 등) |
| **Notion/블로그** | 회고, 기술 의사결정 기록, 트러블슈팅 |
| **발표 자료** | 컨퍼런스, 해커톤, 학교 발표 영상·슬라이드 |
| **공모전·대회** | 수상 이력, 결과물 공개 여부 |

---

## 한국 취준생 특화 프로젝트 유형

### 강한 신호 (채용 담당자 주목)
- **캡스톤 디자인** — 교수 지도 + 실제 문제 해결 + 팀 협업 증명
- **멋쟁이사자처럼·SOPT·DCC 등 IT 연합동아리 프로젝트** — 실전 팀 프로젝트 경험 증명
- **해커톤 수상 프로젝트** — 빠른 실행력 + 결과물 증명 (UNITHON, Junction, SW 마에스트로 등)
- **오픈소스 기여** — PR 이력, 이슈 해결
- **실제 사용자가 있는 서비스** — DAU/MAU, 다운로드 수 등 지표 포함 시 강력

### 보통 신호 (있으면 좋지만 차별화 부족)
- 토이 프로젝트 (TODO 앱, 게시판, 날씨 앱 등) — 너무 흔함
- MOOC 따라 만든 프로젝트 — 독창성 없음
- 배포 없는 코드만 있는 프로젝트

---

## 판정

- **만들 것** → 주차별 마일스톤 포함 계획
- **건너뛸 것** → 이유와 더 나은 대안 제시
- **피벗** → 더 임팩트 있는 변형 아이디어

---

## 80/20 플랜

- **Week 1** → 핵심 기능 + 지표가 있는 MVP, 배포
- **Week 2** → 다듬기 + README + 회고 작성 + 면접 스토리 준비

---

## 면접 팩 구성 요소

승인된 프로젝트에 대해:
1. **원페이저** — 문제 정의, 기술 스택, 핵심 지표, 트러블슈팅 1개
2. **데모** — 접속 가능한 URL 또는 2분 녹화 영상
3. **회고** — 잘된 것, 아쉬운 것, 다시 한다면 바꿀 것
