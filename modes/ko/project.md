# 모드: project — 포트폴리오 프로젝트 평가

만들려는 포트폴리오 프로젝트 아이디어를 6개 차원으로 평가한다.

## 평가 매트릭스 (1-5점)

| 차원 | 가중치 | 5점 | 1점 |
|------|--------|-----|-----|
| 목표 역할 신호 | 25% | JD 핵심 스킬을 직접 증명 | 직무와 무관 |
| 독창성 | 20% | 아무도 안 한 것 | 흔한 토이 프로젝트 |
| 데모 가능성 | 20% | 2분 안에 동작 시연 가능 | 코드만 있고 시각화 불가 |
| 지표 가능성 | 15% | 명확한 측정값 (지연, 비용, 정확도) | 측정 불가 |
| MVP 기간 | 10% | 1주 이내 | 3개월 이상 |
| STAR 스토리 가능성 | 10% | 트레이드오프·의사결정 풍부 | 구현만 있음 |

## 한국 포트폴리오 기준

채용 담당자가 실제로 보는 것:
- **GitHub** — 커밋 이력, README 품질, 코드 가독성
- **배포된 서비스** — 실제 접속 가능한 URL
- **Notion/블로그** — 회고, 기술 의사결정 기록
- **발표 자료** — 컨퍼런스, 사내 발표, 유튜브
- **오픈소스 기여** — PR 이력, 이슈 해결

## 판정

- **만들 것** → 주차별 마일스톤 포함 계획
- **건너뛸 것** → 이유와 더 나은 대안 제시
- **피벗** → 더 임팩트 있는 변형 아이디어 제안

## 80/20 플랜

- **Week 1** → 핵심 지표가 있는 MVP
- **Week 2** → 다듬기 + 면접 팩 작성

## 면접 팩 구성 요소

승인된 프로젝트에 대해:
1. **원페이저** — 문제 정의, 아키텍처, 핵심 지표, 평가 방법
2. **데모** — 접속 가능한 URL 또는 2분 녹화 영상
3. **회고** — 잘된 것, 안 된 것, 대안 검토

## 출력 형식

```markdown
| 차원 | 점수 | 코멘트 |
|------|------|--------|
| 목표 역할 신호 | X/5 | |
| ...

**주차별 계획 (만들 것 판정 시):**
- Week 1: MVP + 핵심 지표
- Week 2: 마무리 + 면접 팩
```
