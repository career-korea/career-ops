# 모드: tracker — 지원 현황 관리

`data/applications.md`를 읽어 지원 현황을 표시하고, 공채 시즌 관리를 포함한다.

---

## 트래커 형식

```markdown
| # | 날짜 | 회사 | 역할 | 점수 | 상태 | PDF | 보고서 | 메모 |
```

## 상태 정의

| 상태 | 의미 |
|------|------|
| `Evaluated` | 공고 평가 완료, 지원 여부 미결정 |
| `Applied` | 서류 제출 완료 |
| `Responded` | 회사·채용담당자로부터 연락 받음 (서류 합격 등) |
| `Interview` | 인적성·코딩테스트·면접 진행 중 |
| `Offer` | 최종 합격 또는 오퍼 수령 |
| `Rejected` | 불합격 (단계 불문) |
| `Discarded` | 후보자 스스로 제외 또는 공고 마감 |
| `SKIP` | 미지원 결정 (점수 낮음 또는 조건 불일치) |

---

## 출력: 기본 현황 요약

```markdown
## 지원 현황 요약 — {날짜}

**전체:** {N}건

| 상태 | 건수 | 비율 |
|------|------|------|
| Evaluated | X | X% |
| Applied | X | X% |
| Interview | X | X% |
| Offer | X | X% |
| Rejected | X | X% |
...

**평균 점수:** X.X/5
**PDF 생성률:** X%

**주목할 항목:**
- 🔴 면접 진행 중: {회사명 목록}
- 🟢 합격 오퍼: {있으면 표시}
- ⏰ 결과 대기 (14일 이상): {Applied 건 중 오래된 것}
```

## 공채 시즌 관리

한국 대기업 공채는 시즌이 있다. 트래커에서 공채 건을 따로 표시한다:

```markdown
## 공채 현황

| 회사 | 시즌 | 마감일 | 상태 | 전형 단계 |
|------|------|--------|------|----------|
| 삼성전자 | 상반기 | 2026-04-30 | Interview | 직무면접 대기 |
| LG전자 | 상반기 | 2026-05-10 | Applied | 서류 심사 중 |
```

**공채 주요 시즌:**
- **상반기**: 보통 3~5월 (지원) → 6~7월 (합격)
- **하반기**: 보통 8~10월 (지원) → 11~12월 (합격)
- **수시채용**: 연중 상시 (스타트업·외국계·IT 기업 중심)

---

## 상태 업데이트

사용자가 상태 변경 요청 시 `data/applications.md`의 해당 행을 직접 수정한다.

예: "카카오 서류 붙었어" → `Applied` → `Interview` 또는 `Responded`

---

## 규칙

- `applications.md`에 신규 행을 직접 추가하지 않는다 — tracker-additions TSV 흐름을 따른다
- 상태 필드에 날짜, 볼드체(`**`), 부가 설명을 넣지 않는다
- 메모 필드만 자유 형식 허용
