# 모드: tracker — 지원 현황 요약

`data/applications.md`를 읽어 지원 현황을 표시한다.

## 트래커 형식

```markdown
| # | 날짜 | 회사 | 역할 | 점수 | 상태 | PDF | 보고서 | 메모 |
```

## 상태 정의

| 상태 | 의미 |
|------|------|
| `Evaluated` | 평가 완료, 지원 여부 미결정 |
| `Applied` | 지원서 제출 완료 |
| `Responded` | 회사 또는 리크루터로부터 연락 받음 |
| `Interview` | 면접 진행 중 |
| `Offer` | 합격 오퍼 수령 |
| `Rejected` | 회사로부터 불합격 통보 |
| `Discarded` | 후보자가 스스로 제외 또는 공고 마감 |
| `SKIP` | 미지원 결정 (점수 낮음 또는 조건 불일치) |

## 출력

통계 포함:

```markdown
## 지원 현황 요약 — {날짜}

**전체:** {N}건
**상태별:**
| 상태 | 건수 | 비율 |
| Applied | X | X% |
| Interview | X | X% |
| ...

**평균 점수:** X.X/5
**PDF 생성률:** X%
**보고서 생성률:** X%

**주목할 항목:**
- 면접 진행 중: {회사명 목록}
- 오퍼 수령: {있으면 표시}
- 후속 연락 필요: {Applied 후 7일 이상 경과}
```

## 상태 업데이트

사용자가 상태 변경 요청 시 `data/applications.md`의 해당 행을 직접 수정한다.

예: "쿠팡 합격" → `Applied` → `Offer` 로 변경

## 규칙

- `applications.md`에 신규 행을 직접 추가하지 않는다 — tracker-additions TSV 흐름을 따른다
- 상태 필드에 날짜, 볼드체(`**`), 부가 설명을 넣지 않는다
- 메모 필드만 자유 형식 허용
