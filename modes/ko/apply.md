# 모드: apply -- 실시간 지원서 작성 보조

후보자가 지원서 화면을 보고 있을 때 질문을 읽고 답변 초안을 만든다. 절대 제출 버튼을 누르지 않는다.

## 흐름

1. 현재 페이지/스크린샷/붙여넣은 질문에서 회사와 역할을 파악한다.
2. `reports/`에서 해당 회사/역할 보고서를 찾는다.
3. 보고서가 있으면 Block B/F/G와 기존 draft answers를 사용한다.
4. 보고서가 없으면 빠른 평가 실행을 제안한다.
5. 모든 질문을 분류하고 답변 초안을 만든다.

## 한국 지원서 질문 유형

- 지원동기
- 직무 적합성
- 주요 프로젝트/성과
- 이직 사유
- 희망 연봉
- 입사 가능일
- 근무 가능 지역/재택 가능 여부
- 병역/비자/취업 가능 여부
- 포트폴리오/깃허브/블로그 링크

## 답변 원칙

- 한국어 질문은 한국어로 답한다.
- 영어 질문은 영어로 답한다.
- CV와 보고서에 있는 사실만 사용한다.
- 과한 겸손이나 과장된 열정 표현을 피한다.
- 300자 제한이 있으면 220-280자 안에 맞춘다.
- 500자 제한이 있으면 350-450자 안에 맞춘다.
- 연봉 질문은 `config/profile.yml`의 목표/최저선을 따른다.
- 위치/비자/입사 가능일은 실제 프로필 값을 따른다.

## 출력 형식

```markdown
## 지원서 답변 초안: {Company} -- {Role}

기준 보고서: #{NNN} | 점수: X.X/5 | 시장: Korea/Global Remote

### 1. {정확한 질문}
> {복사해서 붙여넣을 답변}

### 2. {다음 질문}
> {답변}

---

검토 필요:
- {후보자가 직접 확인해야 할 항목}
```

제출은 후보자가 직접 한다.
