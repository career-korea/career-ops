# 모드: apply — 실시간 지원 어시스턴트

후보자가 Chrome에서 지원서 양식을 작성 중일 때를 위한 인터랙티브 모드. 화면에 있는 내용을 읽고, 해당 공고의 이전 컨텍스트를 불러오며, 양식의 각 질문에 맞춤 답변을 생성한다.

## 요건

- **Playwright 표시(visible) 모드가 최적**: 표시 모드에서는 후보자가 브라우저를 보고 Claude가 페이지와 상호작용할 수 있다.
- **Playwright 없이**: 후보자가 스크린샷을 공유하거나 질문을 직접 붙여넣는다.

## 워크플로우

```text
1. 감지(DETECT)     → 활성 Chrome 탭 읽기 (스크린샷/URL/제목)
2. 식별(IDENTIFY)   → 페이지에서 회사 + 역할 추출
3. 검색(SEARCH)     → reports/의 기존 리포트와 매칭
4. 로드(LOAD)       → 전체 리포트 + 섹션 G(있으면) 읽기
5. 비교(COMPARE)    → 화면의 역할이 평가한 것과 일치? 바뀌었으면 → 알림
6. 분석(ANALYZE)    → 보이는 모든 양식 질문 식별
7. 생성(GENERATE)   → 각 질문마다 맞춤 답변 생성
8. 제시(PRESENT)    → 복붙용으로 정리된 답변 표시
```

## Step 1 — 공고 감지

**Playwright로:** 활성 페이지 스냅샷. 제목, URL, 보이는 내용 읽기.

**Playwright 없이:** 후보자에게 요청:
- 양식 스크린샷 공유 (Read 도구가 이미지를 읽음)
- 또는 양식 질문을 텍스트로 붙여넣기
- 또는 검색할 수 있게 회사 + 역할 말하기

## Step 2 — 컨텍스트 식별 및 검색

1. 페이지에서 회사명과 역할명 추출
2. `reports/`에서 회사명으로 검색 (대소문자 무시 grep)
3. 매칭 시 → 전체 리포트 로드
4. 섹션 G가 있으면 → 이전 답변 초안을 기반으로 로드
5. 매칭 없으면 → 알리고 빠른 auto-pipeline 실행 제안

## Step 3 — 역할 변경 감지

화면의 역할이 평가한 것과 다르면:
- **후보자에게 알림**: "역할이 [X]에서 [Y]로 바뀌었습니다. 재평가할까요, 아니면 새 직함에 맞게 답변을 조정할까요?"
- **조정 시**: 재평가 없이 새 역할에 맞게 답변 조정
- **재평가 시**: 전체 A-F 평가 실행, 리포트 업데이트, 섹션 G 재생성
- **트래커 업데이트**: 해당되면 applications.md의 역할명 변경

## Step 4 — 양식 질문 분석

보이는 모든 질문 식별:
- 자유 텍스트 필드 (커버레터, 왜 이 역할인지 등)
- 드롭다운 (어떻게 알았나, 취업 자격 등)
- 예/아니오 (이주, 비자 등)
- 급여 필드 (범위, 희망액)
- 업로드 필드 (이력서, 커버레터 PDF)

각 질문 분류:
- **섹션 G에 이미 답변됨** → 기존 답변 조정
- **새 질문** → 리포트 + cv.md에서 답변 생성

## Step 5 — 답변 생성

각 질문마다 다음을 따라 답변 생성:

1. **리포트 컨텍스트**: 블록 B의 증거 포인트, 블록 F의 STAR 스토리 사용
2. **이전 섹션 G**: 초안 답변이 있으면 기반으로 삼아 다듬기
3. **"내가 당신을 선택한다" 톤**: auto-pipeline과 동일한 프레임워크
4. **구체성**: 화면에 보이는 JD에서 구체적인 점 인용
5. **career-ops 증거 포인트**: "추가 정보" 필드가 있으면 포함

**출력 형식:**

```text
## [회사] — [역할] 답변

기준: Report #NNN | Score: X.X/5 | 아키타입: [유형]

---

### 1. [정확한 양식 질문]
> [복붙 준비된 답변]

### 2. [다음 질문]
> [답변]

...

---

메모:
- [역할, 변경 등에 대한 관찰]
- [후보자가 검토해야 할 개인화 제안]
```

## Step 6 — 지원 후 (선택)

후보자가 지원서를 제출했다고 확인하면:
1. `applications.md`의 상태를 "Evaluated"에서 "Applied"로 업데이트
2. 리포트의 섹션 G를 최종 답변으로 업데이트
3. 다음 단계 제안: LinkedIn 아웃리치는 `/career-ops contacto`

## 스크롤 처리

양식에 보이는 것보다 질문이 더 많으면:
- 후보자에게 스크롤하고 다른 스크린샷 공유 요청
- 또는 나머지 질문 붙여넣기
- 양식 전체가 처리될 때까지 반복 진행
