# 모드: batch — 공채·묶음 지원 배치 처리

여러 공고를 병렬 워커로 한 번에 평가한다. 공채 시즌 대량 지원 또는 URL 목록이 있을 때 사용한다.

---

## 한국 공채 시즌 특화 전략

한국 대기업·공기업 공채는 **동시다발로 마감**된다. 배치 모드는 이 시기에 특히 유용하다.

### 공채 시즌 배치 활용 흐름

```
공채 시즌 시작 (상반기 3~4월 / 하반기 8~9월)
  ↓
원티드·점핏·사람인·잡코리아에서 관심 공고 URL 수집
  ↓
batch-input.tsv에 URL 목록 추가
  ↓
batch 모드 실행 → 회사별 평가 보고서 + 점수 자동 생성
  ↓
점수 4.0 이상 공고 선별 → 자소서 작성 (jasoseo 모드)
  ↓
우선순위 정하여 지원 (한 번에 너무 많이 지원하지 않음)
```

---

## 아키텍처

```
컨덕터 (헤드 브라우저)
  │
  │  Chrome: 포털 탐색 (원티드·사람인·잡코리아 등)
  │
  ├─ 공고 1: DOM에서 JD + URL 읽기
  │    └─► 헤드리스 워커 → 보고서 .md + 트래커 행
  │
  ├─ 공고 2: 다음 클릭
  │    └─► 헤드리스 워커 → 보고서 .md + 트래커 행
  │
  └─ 완료: tracker-additions 병합 + 우선순위 요약
```

---

## 파일 구조

```
batch/
  batch-input.tsv          # URL 목록
  batch-state.tsv          # 진행 상태 (gitignore)
  batch-runner.sh          # 스탠드얼론 실행
  batch-prompt.md          # 워커 프롬프트
  logs/                    # 공고별 로그 (gitignore)
  tracker-additions/       # 트래커 행 TSV (gitignore)
```

---

## 방식 A: 컨덕터 (헤드 브라우저)

1. `batch-state.tsv` 읽기 → 완료 항목 건너뜀
2. Chrome으로 원티드·사람인·점핏 등 접속
3. 검색 결과 URL 추출 → `batch-input.tsv`에 추가
4. 각 대기 URL에 대해:
   - Chrome에서 공고 열기 → DOM JD 읽기
   - `/tmp/batch-jd-{id}.txt`에 저장
   - 다음 순차 보고서 번호 계산
   - 헤드리스 워커 실행:
     ```bash
     claude -p "공고 처리: URL: {url}. JD: /tmp/batch-jd-{id}.txt. 보고서번호: {num}"
     ```
   - `batch-state.tsv` 업데이트
5. 페이지네이션 처리 → 완료 시 병합

---

## 방식 B: 스탠드얼론

```bash
batch/batch-runner.sh [옵션]
```

옵션:
- `--dry-run` — 실행 없이 목록만 확인
- `--retry-failed` — 실패 항목만 재시도
- `--parallel N` — N개 병렬 워커
- `--max-retries N` — 항목당 최대 시도 횟수

## 공채 시즌 배치 완료 후 요약 형식

```markdown
## 공채 배치 평가 결과 — {날짜}

**총 평가:** {N}건
**4.0 이상 (지원 추천):** {N}건
**3.5~3.9 (조건부):** {N}건
**3.5 미만 (비추천):** {N}건

### 지원 추천 공고

| # | 회사 | 역할 | 점수 | 마감일 | 자소서 필요 |
|---|------|------|------|--------|------------|
```

## 한국 포털 주의사항

| 포털 | 주의점 |
|------|--------|
| 원티드 | JS 렌더링, 로그인 팝업 있을 수 있음 |
| 사람인·잡코리아 | 공고 중복 게재 많음 — 중복 제거 필수 |
| 점핏 | 프로그래머스 계정 연동 |
| 대기업 자체 채용 사이트 | 삼성·LG·현대 등은 자체 ATS, 스크래핑 제한 있음 |
| 공공기관 | 나라일터·워크넷 — 별도 로그인 필요 |

로그인 필요 공고는 `[!]`로 표시 후 수동 처리 요청.

---

## 재개 가능성

- 중단 시 재실행 → `batch-state.tsv` 읽기 → 완료 건 건너뜀
- 잠금 파일로 중복 실행 방지
- 각 워커 독립 실행 (한 공고 실패해도 나머지 계속 진행)
