# 모드: jasoseo — 자기소개서 AI 생성

사용자가 공고(URL 또는 텍스트)와 지원 동기를 주면 자소서 항목별 초안을 작성한다.

## Step 0 — 입력 확보

다음 정보를 수집한다:
1. **공고**: URL 또는 JD 텍스트 (없으면 회사명+직무명만으로 진행)
2. **cv.md** + **config/profile.yml** + **modes/_profile.md** 읽기
3. 글자수 제한 확인 (사용자가 명시하지 않으면 항목당 500자로 기본 설정)
4. **원본 자소서**(기존에 작성된 답변) 존재 여부 확인. 입력에 `## 원본 자소서` 섹션이 있으면 아래 **"첨삭 모드"**를 따른다 — Step 1~4(백지 초안 생성)는 건너뛴다. 없으면 평소대로 Step 1부터 진행.

## 첨삭 모드 (원본 자소서가 있을 때)

원본 자소서의 문항과 답변을 JD 대비 채점하고, 문항별로 감점 요인과 재작성본을 제시한다. 절대 수치를 지어내지 않는다 — cv.md에 없는 성과나 숫자는 넣지 않는다.

### 1. ATS 요약 블록 출력 (최우선)

다른 내용보다 **반드시 먼저** 아래 형식의 코드 블록을 출력한다. 프론트엔드가 이 블록을 파싱해 시각적 대시보드를 렌더링하므로 형식을 정확히 지킨다. 자소서 전체(모든 문항 합산)에 대한 총평이다.

```ats-summary
score: [0-100 정수. 문항별 감점 총합을 100점에서 뺀 값]
grade: [탁월|경쟁력 있음|개선 필요|보완 필요|개선 시급|재검토 필요|심각한 불일치]
legitimacy: High Confidence
archetype: [JD 기반 감지된 아키타입]
match_keywords: [JD와 자소서 모두에 있는 키워드, 쉼표 구분, 최대 8개]
missing_keywords: [JD에 있으나 자소서에 없는 키워드, 쉼표 구분, 최대 8개]
highlight_skills: [자소서에서 더 강조해야 할 스킬, 쉼표 구분, 최대 5개]
deductions: [전체 감점 항목 요약, 파이프(|)로 구분. 예: [-8점] 정량적 성과 수치 없음|[-5점] 핵심 키워드 누락]
summary: [ATS 관점 한 줄 진단. 50자 이내]
hooks: [원본을 대체할 만한 첫 문장 후보 3개, 파이프(|)로 구분]
```

`legitimacy`는 이 컨텍스트에서 의미가 없으므로 항상 `High Confidence`로 고정한다.

### 2. 문항별 첨삭

원본에 있는 문항마다 (여러 개면 각각) 이 순서로, 반드시 `### ` 헤더를 정확히 이 형식으로 사용한다 (프론트엔드가 이 헤더로 탭을 나눈다):

`### 문항 N — 해설본`

- 원본 답변에서 발견된 감점 요인 2~4개를 나열한다. 각 요인은:
  - `**[-N점] {요인명}**` (요인명 예: 전략 부재, 수치화 부족, 직무 본질에 대한 통찰 부재, 진부한 역량 정의)
  - `> {원본에서 인용한 문제 문장}` (원문 그대로 인용, blockquote로)
  - `💡 개선 방안: {구체적으로 무엇을 바꿔야 하는지}`
- 마지막에 감점 총합과 1~2문장 총평.

`### 문항 N — 완성본`

- **JD 핵심 키워드 삽입**, **전략적 방향으로 개선**, **구체적 성과 수치 보강** 등 적용한 변경 카테고리별로 2~4개의 Before→After 문장 비교를 보여준다:
  - `❌ {원본 문장}`
  - `✅ {개선된 문장}`
  - (색상 하이라이트는 쓰지 않는다 — 바뀐 표현은 **볼드**로 강조한다)
- 마지막에 문항 전체를 재작성한 완성된 답변 전문을 싣는다. 이게 사용자가 그대로 복사해 쓸 수 있는 최종본이다.

## Step 1 — 공고 분석

JD에서 추출:
- 핵심 역량 키워드 (3~5개)
- 회사 핵심 가치 / 인재상
- 직무 주요 업무
- 이 회사가 가장 중요하게 보는 것

## Step 2 — 자소서 항목별 생성

아래 4개 항목 작성. 각 항목은:
- **STAR 구조** 사용: 상황(S) → 과제(T) → 행동(A) → 결과(R)
- cv.md의 실제 경험과 수치 인용 (수치 없으면 구체적 행동 묘사)
- 공고 키워드 자연스럽게 2~3개 포함
- 마지막에 글자수 표시

### 1. 성장과정 및 지원동기
- 지원자의 경험 → 이 직무·회사로 이어지는 서사 구성
- "왜 지금 이 회사인가"를 설득력 있게 마무리

### 2. 직무 역량 및 경험
- cv.md의 가장 관련 경험 1~2개를 STAR로 전개
- 숫자·결과 중심 서술
- JD 핵심 역량과 직접 연결

### 3. 입사 후 포부
- 단기(1년) / 중기(3년) 목표 제시
- 회사의 방향성과 연결
- 구체적 기여 방식 언급

### 4. 성격의 장단점
- 장점: 직무에 도움되는 특성 + 근거 경험
- 단점: 개선 중인 부분 + 극복 노력 (약점을 강점으로 전환하는 프레임)

## Step 3 — 맞춤화 체크

각 항목 작성 후:
- [ ] 회사명 직접 언급 여부
- [ ] 글자수 제한 준수 여부
- [ ] 수치/근거 포함 여부
- [ ] 진부한 표현("열정 있는", "최선을 다하는") 제거 여부

## Step 4 — 출력 형식

```
## [회사명] [직무명] 자기소개서

---
### 1. 성장과정 및 지원동기 (XXX자)
[내용]

---
### 2. 직무 역량 및 경험 (XXX자)
[내용]

---
### 3. 입사 후 포부 (XXX자)
[내용]

---
### 4. 성격의 장단점 (XXX자)
[내용]

---
**작성 메모:**
- 사용된 핵심 키워드: [목록]
- 보완 추천: [있으면 1~2줄]
```

## 규칙

- 절대 수치를 지어내지 않는다 — cv.md에 없으면 "구체적 수치는 추후 보완" 표시
- 공허한 표현 금지: "열심히", "최선을 다해", "성장하고 싶습니다"
- 글자수가 초과하면 핵심 내용 유지하며 압축
- 사용자가 수정 요청하면 해당 항목만 재작성
