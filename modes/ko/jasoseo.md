# 모드: jasoseo — 자기소개서 AI 생성

사용자가 공고(URL 또는 텍스트)와 지원 동기를 주면 자소서 항목별 초안을 작성한다.

## Step 0 — 입력 확보

다음 정보를 수집한다:
1. **공고**: URL 또는 JD 텍스트 (없으면 회사명+직무명만으로 진행)
2. **cv.md** + **config/profile.yml** + **modes/_profile.md** 읽기
3. 글자수 제한 확인 (사용자가 명시하지 않으면 항목당 500자로 기본 설정)

## Step 1 — 공고 분석

JD에서 추출:
- 핵심 역량 키워드 (3~5개)
- 회사 핵심 가치 / 인재상
- 직무 주요 업무
- 이 회사가 가장 중요하게 보는 것

## Step 2 — 자소서 항목별 생성

아래 4개 항목 작성. 각 항목은:
- **STAR 구조** 사용: 상황(S) → 과제(T) → 행동(A) → 결과(R)
- cv.md의 실제 경험과 수치 인용 (수치 없으면 구체적 행동 묘사)
- 공고 키워드 자연스럽게 2~3개 포함
- 마지막에 글자수 표시

### 1. 성장과정 및 지원동기
- 지원자의 경험 → 이 직무·회사로 이어지는 서사 구성
- "왜 지금 이 회사인가"를 설득력 있게 마무리

### 2. 직무 역량 및 경험
- cv.md의 가장 관련 경험 1~2개를 STAR로 전개
- 숫자·결과 중심 서술
- JD 핵심 역량과 직접 연결

### 3. 입사 후 포부
- 단기(1년) / 중기(3년) 목표 제시
- 회사의 방향성과 연결
- 구체적 기여 방식 언급

### 4. 성격의 장단점
- 장점: 직무에 도움되는 특성 + 근거 경험
- 단점: 개선 중인 부분 + 극복 노력 (약점을 강점으로 전환하는 프레임)

## Step 3 — 맞춤화 체크

각 항목 작성 후:
- [ ] 회사명 직접 언급 여부
- [ ] 글자수 제한 준수 여부
- [ ] 수치/근거 포함 여부
- [ ] 진부한 표현("열정 있는", "최선을 다하는") 제거 여부

## Step 4 — 출력 형식

```
## [회사명] [직무명] 자기소개서

---
### 1. 성장과정 및 지원동기 (XXX자)
[내용]

---
### 2. 직무 역량 및 경험 (XXX자)
[내용]

---
### 3. 입사 후 포부 (XXX자)
[내용]

---
### 4. 성격의 장단점 (XXX자)
[내용]

---
**작성 메모:**
- 사용된 핵심 키워드: [목록]
- 보완 추천: [있으면 1~2줄]
```

## 규칙

- 절대 수치를 지어내지 않는다 — cv.md에 없으면 "구체적 수치는 추후 보완" 표시
- 공허한 표현 금지: "열심히", "최선을 다해", "성장하고 싶습니다"
- 글자수가 초과하면 핵심 내용 유지하며 압축
- 사용자가 수정 요청하면 해당 항목만 재작성
