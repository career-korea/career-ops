# 모드: patterns — 탈락 패턴 분석

전체 지원 이력을 분석해 반복되는 탈락 원인을 찾고 전략 개선 방향을 제시한다.

---

## 입력

- `data/applications.md` — 지원 트래커
- `reports/` — 개별 평가 보고서
- `config/profile.yml` — 후보자 프로필
- `modes/_profile.md` — 목표 역할 및 아키타입

---

## 최소 데이터 기준

분석 전 확인: `Evaluated` 이후 단계가 **5건 이상**인가?

없으면:
> "아직 데이터가 부족합니다 — {N}/5건만 진행됐습니다. 지원을 더 진행하고 다시 실행해주세요."

---

## Step 1 — 분석 스크립트 실행

```bash
node analyze-patterns.mjs
```

---

## Step 2 — 한국 채용 구조 기반 탈락 원인 분석

단순 통계 외에 다음 관점으로 추가 분석한다:

### 서류 탈락 원인 진단

| 탈락 원인 | 진단 질문 |
|----------|---------|
| 스펙 미달 | 학점, 어학점수, 자격증이 기준치 미달인가? |
| 자소서 문제 | 지원동기가 구체적인가? 직무 연관성이 드러나는가? |
| 포트폴리오 부재 | 관련 프로젝트·인턴·공모전 경험이 없는가? |
| 직무 불일치 | 전공·경험이 직무 요구와 너무 멀리 있는가? |
| 경쟁 과다 | 지원자가 몰리는 유명 공채에 집중하고 있는가? |

### 면접 탈락 원인 진단

| 탈락 원인 | 진단 질문 |
|----------|---------|
| 지원동기 부족 | "왜 이 회사인가"에 구체적으로 답하지 못했는가? |
| STAR 스토리 부재 | 경험을 구조적으로 전달하지 못했는가? |
| 직무 이해 부족 | 지원 직무의 실제 업무를 충분히 파악했는가? |
| 인적성·코딩 통과 실패 | 준비 없이 응시했는가? |
| 자신감·전달력 | 내용은 좋지만 전달이 약했는가? |

## Step 3 — 보고서 작성

`reports/pattern-analysis-{YYYY-MM-DD}.md`에 저장:

```markdown
# 패턴 분석 — {YYYY-MM-DD}

**분석 건수:** {N}건 | **기간:** {시작} ~ {종료}
**결과:** 긍정 {N} / 부정 {N} / 자기제외 {N} / 대기 {N}

---

## 전환 퍼널

| 단계 | 건수 | 비율 |
|------|------|------|
| 서류 제출 | X | 100% |
| 서류 합격 | X | X% |
| 면접 진행 | X | X% |
| 최종 합격 | X | X% |

## 점수 vs 결과

| 결과 | 평균 점수 | 건수 |
|------|---------|------|

## 회사 유형별 성과

대기업 / 중견기업 / 스타트업 / 공기업 별 지원 건수와 합격률

## 주요 탈락 패턴

1. {가장 빈번한 서류 탈락 원인}
2. {면접 탈락 패턴}
3. {자기제외 원인}

## 권장 점수 기준

데이터 기반 최소 지원 점수: {X.X}/5

## 개선 추천 (상위 3개)

1. **[고임팩트]** {액션} — {근거}
2. **[중임팩트]** {액션} — {근거}
3. **[중임팩트]** {액션} — {근거}
```
