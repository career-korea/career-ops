# 모드: patterns — 탈락 패턴 분석

전체 지원 이력을 분석해 패턴을 찾고 전략 개선 방향을 제시한다.

## 입력

- `data/applications.md` — 지원 트래커
- `reports/` — 개별 평가 보고서
- `config/profile.yml` — 후보자 프로필
- `modes/_profile.md` — 목표 역할 및 아키타입

## 최소 데이터 기준

분석 실행 전 확인: `Evaluated` 이후 단계(Applied, Interview, Offer, Rejected 등)가 **5건 이상** 있는가?

없으면:
> "아직 데이터가 부족합니다 — {N}/5건만 진행됐습니다. 지원을 더 진행하고 다시 실행해주세요."

## Step 1 — 분석 스크립트 실행

```bash
node analyze-patterns.mjs
```

JSON 결과를 파싱한다.

## Step 2 — 보고서 작성

`reports/pattern-analysis-{YYYY-MM-DD}.md`에 저장.

```markdown
# 패턴 분석 — {YYYY-MM-DD}

**분석 건수:** {N}건
**기간:** {시작} ~ {종료}
**결과 분포:** 긍정 {N}, 부정 {N}, 자기제외 {N}, 대기 {N}

---

## 전환 퍼널

| 단계 | 건수 | 비율 |
|------|------|------|
| 평가 완료 | X | X% |
| 지원 | X | X% |
| 면접 | X | X% |
| 합격 | X | X% |

## 점수 vs 결과

| 결과 | 평균 점수 | 최저 | 최고 | 건수 |
|------|---------|------|------|------|

## 아키타입별 성과

어떤 직무 유형에서 면접/합격이 많이 나왔는지.
가장 성과 좋은 아키타입과 가장 낮은 아키타입 강조.

## 주요 블로커

지원 탈락/자기제외의 반복 원인 빈도표.
(위치 조건 불일치, 기술 스택 갭, 포괄임금, 연봉 불일치 등)

## 권장 점수 기준

데이터 기반 최소 지원 점수 및 근거.

## 액션 추천

상위 3개 개선 방향 (임팩트 높은 순서).
```

## Step 3 — 요약 제시

```
패턴 분석 완료 ({N}건, {기간})

핵심 발견:
- {발견 1}
- {발견 2}
- {발견 3}

전체 보고서: reports/pattern-analysis-{날짜}.md
```

## Step 4 — 추천사항 적용 제안

> "다음 중 적용할 항목이 있으신가요?
> - `portals.yml` 지역 필터 조정
> - `_profile.md` 목표 아키타입 수정
> - `config/profile.yml` 점수 기준 업데이트"

사용자 동의 시:
- 포털 필터 변경 → `portals.yml` 수정
- 아키타입 변경 → `modes/_profile.md` 수정 (절대 `_shared.md` 아님)

## 결과 분류 기준

| 상태 | 분류 |
|------|------|
| Interview, Offer, Responded, Applied | **긍정** |
| Rejected, Discarded | **부정** |
| SKIP | **자기제외** |
| Evaluated | **대기** |
