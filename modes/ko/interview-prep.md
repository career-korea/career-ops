# 모드: interview-prep — 회사별 면접 준비

지원 회사+역할에 맞춘 면접 인텔리전스 문서를 생성한다. 점수 4.0 이상이거나 사용자가 면접 단계 진입 시 실행한다.

## 입력

1. **회사명 + 역할** (필수)
2. `reports/`의 평가 보고서 (있으면 읽기 — Block B/F 참조)
3. `interview-prep/story-bank.md` (기존 STAR 스토리)
4. `cv.md` + `config/profile.yml` + `modes/_profile.md`

## Step 1 — 리서치

### 보상/처우
- 잡플래닛·크레딧잡·블라인드에서 연봉 범위 확인
- 성과급(PI/PS), 스톡옵션, 사이닝 보너스 여부
- 포괄임금 여부 및 실질 시급 환산

### 채용 프로세스
- 전형 단계 수와 예상 소요 기간
- 인적성 시험 여부 (GSAT/LG Way Fit/HMAT/CJ CAT 등)
- 코딩 테스트 플랫폼 (프로그래머스/코딩테스트/사내 등)
- PT 면접·토론 면접 포함 여부
- 잡플래닛·블라인드에서 실제 면접 후기 확인

### 회사 문화/기술
- 기술 블로그 최근 포스트 주제
- 사용 기술 스택 (채용공고 + GitHub)
- 블라인드에서 현직자가 자주 언급하는 특징

## Step 2 — 전형 단계 정리

```markdown
## 전형 개요
- **단계:** {N}단계, 약 {X}주 소요 예상
- **형식:** {예: 서류 → 인적성 → 1차 직무면접 → 2차 임원면접}
- **코딩테스트:** {있음/없음, 플랫폼}
- **인적성:** {있음/없음, 종류}
- **알려진 특이사항:** {예: "화상면접 진행", "PT 면접 포함"}
- **출처:** {링크}
```

## Step 3 — 단계별 면접 유형 분류

| 단계 | 면접관 | 평가 포인트 |
|------|--------|------------|
| 인사/채용 담당 | HR, 리크루터 | 지원동기, 이직 사유, 희망연봉, 입사 가능일 |
| 직무 면접 | 현업 팀장/팀원 | 기술 깊이, 프로젝트 경험, 문제 해결 방식 |
| 임원 면접 | 임원·부서장 | 가치관, 성장 의지, 조직 적합성 |
| 혼합 패널 | 복수 면접관 | 위 항목 통합 |

## Step 4 — 단계별 예상 질문 + 답변 가이드

### 인사/채용 담당 단계

최소 준비 항목:
- **1분 자기소개** — `modes/_profile.md` 서사 기반, 지원 역할과 연결
- **지원동기** — 회사 특정 사업/기술/문화 요소와 cv.md 경험 연결
- **이직 사유** — 방어적이지 않게, 성장 지향으로 프레임
- **희망 연봉** — `config/profile.yml`의 목표 기준. 시장 데이터 근거 제시
- **입사 가능일** — 프로필 실제 값 사용
- **포괄임금/야근 질문** — `modes/_profile.md` 위치 정책 참조

### 직무 면접 단계

기술/역할별 예상 질문 (JD 기반 추론):

| # | JD 요구사항 | 예상 질문 | cv.md 근거 | 답변 각도 |
|---|------------|----------|-----------|---------|

STAR+R 스토리 우선순위:
- `interview-prep/story-bank.md`에서 관련 스토리 매핑
- 없으면 cv.md에서 새 스토리 발굴 제안

한국 직무 면접 필수 준비:
- "왜 우리 회사인가" (경쟁사 대비 이유)
- "팀 내 갈등·협업 어려움 경험"
- "가장 어려웠던 기술적 문제와 해결 방식"
- "프로젝트 실패 경험과 교훈"

### 임원 면접 단계

- **회사 비전과 나의 방향 연결** — 최근 회사 뉴스/전략과 연결
- **10년 후 목표** — 회사 내 성장과 결부
- **조직 생활 가치관** — 협업, 보고, 의사결정 방식
- **역질문** — 팀의 현재 과제, 올해 가장 중요한 목표

## Step 5 — STAR 스토리 매핑

`interview-prep/story-bank.md`와 연결:

| # | 단계 | 예상 질문/주제 | 최적 스토리 | 적합도 | 갭 |
|---|------|--------------|-----------|--------|---|
| 1 | 직무 | ... | [스토리 제목] | 강/보통/없음 | |

갭 발견 시: "'{주제}' 관련 스토리가 없습니다. cv.md의 '{경험}'을 STAR+R로 발전시킬 수 있습니다."

## Step 6 — 기술 준비 체크리스트

JD와 리서치 기반 (최대 10개):

```markdown
- [ ] {기술/주제} — 이유: "{JD 요구사항 또는 블라인드 후기 근거}"
```

## 출력

`interview-prep/{company-slug}-{role-slug}.md`에 저장:

```markdown
# 면접 준비: {회사명} — {역할}

**URL:** {채용공고 URL}
**평가 보고서:** {reports/ 링크 또는 없음}
**리서치 일자:** {YYYY-MM-DD}
**소스:** 잡플래닛 후기 {N}건, 블라인드 {N}건, 기타
**전형 단계:** {확인된 단계}
```

## 규칙

- 면접 질문을 만들 때 출처가 없으면 반드시 `[JD 기반 추론]`으로 표기
- 잡플래닛·블라인드 통계를 추측하지 않는다
- 모든 데이터에 출처 표기
- 보상 데이터가 없으면 "확인 불가"로 명시
- 전달할 수 없는 부분이 있어도 부분적 출력이 출력 안 함보다 낫다
